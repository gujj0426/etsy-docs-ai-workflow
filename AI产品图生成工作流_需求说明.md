# 网店AI产品图生成工作流
## 需求说明文档 v1.0

---

## 一、项目背景

### 1.1 业务概述

- **主营平台**：Etsy（个性化定制饰品）
- **主营产品**：领带夹（双面定制）、袖扣、狗牌（宠物纪念品）、其他个性化饰品
- **业务特点**：个性化定制为主，客户需提供照片，设计师进行AI图生图处理后用于Listing展示

### 1.2 现有系统

| 系统 | 用途 | 状态 |
|------|------|------|
| 腾讯智能表格「产品开发进度管理」 | 管理产品开发全链路进度 | 已上线 |
| 派单表格（每日生成） | 记录当日订单和分配信息 | 现有业务流程 |

### 1.3 本项目目标

搭建一套**自动化的AI产品图生成工作流**，实现：
- 派单 → AI自动生成粗略图 → 设计师精修 → 打包回传 → 派单审单
- 全链路任务状态可追踪、可管理
- 不同产品类型自动调用最合适的AI模型

## 二、业务流程

### 2.1 完整链路

```
[派单人员]
  生成当日派单表格
  ↓
  按产品类型分配设计师（领带夹/袖扣/狗牌/其他）
  ↓
  将客户照片按订单编号命名，放入「待生成」文件夹
─────────────────────────────────────────────
[AI自动生成环节]
  Python脚本监控文件夹
  ↓
  读取产品类型，自动匹配AI模型
  ↓
  调用对应API生成粗略图
  ↓
  图片存入「已生成」文件夹（以订单编号命名）
─────────────────────────────────────────────
[设计师]
  收到「已生成」文件夹中的图片通知
  ↓
  精修图片（PS/AI精修）
  ↓
  将精修图放回「待精修」文件夹
─────────────────────────────────────────────
[派单人员]
  收到精修完成通知
  ↓
  审单确认
  ↓
  图片归档，完成该订单
```

### 2.2 角色与职责

| 角色 | 职责 |
|------|------|
| 派单人员 | 生成派单表格、分配任务、审单归档 |
| 设计师（凡歌） | 选品采购相关设计 |
| 设计师（可心） | 拍摄+图片精修 |
| AI（自动） | 根据产品类型自动调用模型生成粗略图 |

## 三、功能需求

### 3.1 文件夹管理

建立标准化的文件夹结构：

```
/AI产品图生成/
├── 待生成/          # 派单人员放入客户原图，按订单编号命名
│   ├── {订单编号}_客户姓名.jpg
│   ├── {订单编号}_客户姓名.png
├── 生成中/          # 脚本正在处理中的任务
├── 已生成/          # AI粗略图输出，设计师取走
│   ├── {订单编号}_客户姓名_generated.png
├── 待精修/          # 设计师精修后放回
├── 已完成/          # 最终版，派单取走审单归档
└── 原始照片备份/    # 保留客户原始照片备查
```

**命名规范**：
- 格式：`{订单编号}_{客户姓名}.{后缀}`
- 示例：`ORD20250415_001_张三.jpg`
- 脚本自动从文件名提取订单编号

### 3.2 AI模型路由（自动选模型）

根据产品类型自动调用不同模型：

| 产品类型 | 推荐模型 | 原因 | 备选模型 |
|---------|---------|------|---------|
| 领带夹 | Kolors（硅基流动） | 金属光泽、精细纹理表现好 | 即梦 |
| 袖扣 | Kolors（硅基流动） | 同上 | 即梦 |
| 狗牌 | Kolors（硅基流动） | 宠物饰品场景理解好 | 即梦 |
| 其他 | Kolors（硅基流动） | 默认 | 即梦 |

**模型切换逻辑**：Python脚本中通过产品类型字段判断调用哪个API，支持按需增减。

### 3.3 任务状态管理

在腾讯智能表格中新建「图生图任务」工作表：

| 字段 | 类型 | 说明 |
|------|------|------|
| 订单编号 | 文本 | 唯一标识 |
| 客户姓名 | 文本 | - |
| 产品类型 | 单选 | 领带夹/袖扣/狗牌/其他 |
| 原始照片 | 文本 | 文件路径 |
| 派单时间 | 日期时间 | - |
| AI生图状态 | 单选 | 待生成/生成中/已生成/失败 |
| AI生图模型 | 文本 | 记录用了哪个模型 |
| 生成时间 | 日期时间 | 完成后自动记录 |
| 设计师 | 文本 | 负责精修的人 |
| 精修状态 | 单选 | 待精修/精修中/已完成 |
| 回传时间 | 日期时间 | - |
| 派单审单 | 单选 | 待审/通过/需返工 |
| 备注 | 文本 | - |

**AI生图状态枚举**：
- `待生成`：已分配，等待脚本处理
- `生成中`：脚本正在调用API
- `已生成`：AI粗略图完成，等待设计师取走
- `失败`：API调用异常，需人工介入

### 3.4 通知机制

| 事件 | 通知方式 | 接收人 |
|------|---------|-------|
| 新任务到达「已生成」 | 微信/钉钉机器人推送 或 腾讯表格变更通知 | 设计师 |
| AI生图失败 | 同上 | 派单人员 |
| 精修完成 | 同上 | 派单人员 |
| 全部完成 | 同上 | 派单人员 |

## 四、技术方案

### 4.1 技术栈选择

| 层级 | 工具 | 决策 | 状态 |
|------|------|------|------|
| 图生图API | 硅基流动（Kolors） | 国内直连，免费额度，按量计费 | 待注册 |
| 图生图API（备选） | 即梦（火山引擎） | 需VPN，效果好 | 待评估 |
| 任务编排 | Python脚本 | 最灵活，最快落地 | 待开发 |
| 任务状态管理 | 腾讯智能表格 | 复用现有平台 | 待新建工作表 |
| 文件存储 | 本地文件夹 | 免费，可搭配NAS | 现有 |
| 通知 | 微信/钉钉机器人 | 免费 | 可选后续接入 |

### 4.2 平台对比（决策参考）

#### 硅基流动（硅云）
- **优点**：国内直连，无需VPN，Kolors模型免费额度高，按量计费便宜
- **缺点**：不支持FLUX.1，Kolors对某些精细材质表现可能不如FLUX
- **适合场景**：快速落地、日常订单处理

#### 即梦（火山引擎）
- **优点**：即梦4.0模型效果好，字节跳动出品
- **缺点**：需要VPN访问
- **适合场景**：对图片质量要求高的订单

#### Replicate
- **优点**：FLUX.1开源最强模型
- **缺点**：海外平台，需VPN，费用较高
- **适合场景**：最高质量要求

**建议**：优先使用硅基流动Kolors跑通流程，效果满足需求则不再切换；如质量不够再增加即梦或Replicate作为备选。

### 4.3 Python脚本核心逻辑

```
监控「待生成」文件夹（watchdog）
    ↓
发现新文件 → 解析订单编号 + 产品类型
    ↓
更新腾讯表格状态 →「生成中」
    ↓
根据产品类型选择模型
    ↓
调用对应API（硅基流动/即梦）
    ↓
下载生成图片 → 保存到「已生成」
    ↓
移动原文件到「生成中」（处理中标记）
    ↓
更新腾讯表格状态 →「已生成」
    ↓
发送通知给设计师
```

**表格链接**：https://docs.qq.com/space/DS0hhVFRpd1ZRRmxK?resourceId=KxovjYqCxYnM&mode=wiki_mode

**空间ID**：KHaTTiwVQFlJ
**文件夹ID**：KnYdrMdDfgWY
**智能表格file_id**：KxovjYqCxYnM

**工作表**：选品采购（sheet_id: pyaJxh）— 合并选品采购+拍摄，统一管理

**字段（共12列）**：
产品名称 → 产品分类 → 角色 → 负责人 → 优先级 → **开发状态** → **进度** → 阻碍 → 计划开始 → 计划完成 → 实际完成 → 备注

**开发状态（10阶段）**：
```
待开始 → 市场调研 → 选品评估 → 供应商筛选 → 样品确认
→ 已采购 → 等待样品 → 拍摄中 → 修图中 → Listing上传 → 已完成
```

**进度公式**（单元格手动设置一次，整列自动计算）：
```
=SWITCH([开发状态],
  "待开始",0,
  "市场调研",0.2,
  "选品评估",0.3,
  "供应商筛选",0.4,
  "样品确认",0.55,
  "已采购",0.6,
  "等待样品",0.65,
  "拍摄中",0.75,
  "修图中",0.85,
  "Listing上传",0.95,
  "已完成",1
)
```
进度列格式设为「百分比」，小数位0。

**优先级枚举**：🔴 高 / 🟡 中 / 🟢 低

**负责人预设**：选品采购 → 凡歌，拍摄 → 可心

**甘特图视图**：腾讯表格支持甘特图视图，建议手动在web端创建，每个阶段一行，以计划开始/计划完成作为甘特条起止点，阶段完成日期作为里程碑节点。
