## 四、AI生图触发规则（核心逻辑）

### 4.1 触发条件

满足以下任一条件 → 进入AI生图流程：

| 触发条件 | 触发字段 | 说明 |
|---------|---------|------|
| **宠物头像** | 设计风格 = `宠物头像(见附图)` | 需用客户提供的宠物照片生图 |
| **人物头像** | 设计风格 = `人物头像(见附图)` | 需用客户提供的人物照片生图 |
| **预览图** | 是否预览 = `Y` | 客户要求预览效果 |
| **见附图** | 设计风格 = `见附图` | 有附件参考图需处理 |
| **相盒照片** | 设计风格 = `相盒照片` | 相盒类定制 |

### 4.2 模型选择规则

| 条件 | 使用模型 | 原因 |
|------|---------|------|
| **颜色 = 黑色** | 即梦（Jimeng） | 黑色产品光照难度高，即梦对深色材质表现更好 |
| 颜色 = 其他（银色/金色/玫瑰金/白色等） | Kolors（硅基流动） | 通用场景，效果稳定 |

## 七、AI生图工作流（重构版）

### 7.1 完整业务流程

```
[派单人员 填写「3、美国仓库出库明细」]
  → 填写设计风格 = 宠物头像/人物头像/预览图 等
  → 填写颜色 = 黑色（触发即梦）/ 其他（触发Kolors）
  → 填写刻录信息 = 客户个性化需求描述
  → 填写设计师 = 小如/之琳/凡歌/羽桐
  ↓
[Python脚本 定时轮询表格]
  → 检测到新触发任务（设计风格 in [宠物头像, 人物头像, 见附图, 相盒照片]）
  → 识别：订单编号 + 产品类型 + 颜色 + 设计师
  ↓
[AI自动生图]
  → 颜色=黑色 → 调用即梦API（需VPN）
  → 颜色=其他 → 调用Kolors API（硅基流动，国内直连）
  → 输入：客户照片 + 刻录信息描述 + 产品类型prompt
  → 输出：电商展示级白底产品图
  ↓
[图片分发]
  → 生成的图片按订单编号命名
  → 存入对应设计师的「已生成」文件夹
  → 通知设计师取走精修
  ↓
[设计师精修]
  → 精修后放回「待审单」文件夹
  ↓
[派单审单]
  → 确认图片效果
  → 完成出库流程
```

### 7.2 模型路由逻辑（Python脚本）

```
IF 颜色 == "黑色":
    调用即梦API (火山引擎)
    # prompt = "黑色{产品名称}，{设计风格}，{刻录信息}，白色背景，电商展示"
ELSE:
    调用Kolors API (硅基流动)
    # prompt = "{颜色}{产品名称}，{设计风格}，{刻录信息}，白色背景，电商展示"
```

### 7.3 文件夹结构

```
/AI产品图生成/
├── 待生成/              # 客户原图，按订单编号命名
├── 生成中/              # 脚本处理中
├── 已生成_Kolors/        # Kolors生成的黑底/白底图
├── 已生成_Jimeng/        # 即梦生成的黑色产品图
├── 待精修/              # 设计师精修
├── 已完成/              # 最终版，派单审单
└── 原始照片备份/         # 客户原始照片备份
```
